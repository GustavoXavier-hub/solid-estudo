# Interface Segregation Principle (ISP)

## Definição
Nenhum cliente deve ser forçado a depender de métodos que não utiliza.

## Problema
Interfaces grandes forçam classes a implementar métodos desnecessários.

## Solução
Criar interfaces menores e mais específicas.

## Exemplos
- `bad-example.ts`: Exemplo violando o ISP
- `good-example.ts`: Exemplo seguindo o ISP
