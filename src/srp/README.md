# Single Responsibility Principle (SRP)

## Definição
Uma classe deve ter apenas uma razão para mudar, ou seja, deve ter apenas uma responsabilidade.

## Problema
Quando uma classe tem múltiplas responsabilidades, mudanças em uma responsabilidade podem afetar as outras.

## Solução
Separar as responsabilidades em classes distintas.

## Exemplos
- `bad-example.ts`: Exemplo violando o SRP
- `good-example.ts`: Exemplo seguindo o SRP
